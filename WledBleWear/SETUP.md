> Assumes familiarity with programming but **no prior WearOS/Android Studio experience**.
> All steps are Windows-specific unless noted.

## 1 — Create the skeleton project in Android Studio

Android Studio generates `gradle-wrapper.jar` (a binary that cannot be delivered as source) and the correct Gradle wrapper during project creation. You **must** do this step; do not skip to "replace files".

1. Open Android Studio → **New Project**.
2. Select template **"Wear OS → Blank Activity"**.
3. Fill in:
   - **Name:** `WledBleWear`
   - **Package name:** `com.jpchurchouse.wledblewear`
   - **Save location:** your preferred path (e.g. `C:\Users\Jamie\Git\WledBleWear`)
   - **Language:** Kotlin
   - **Minimum SDK:** API 26 (Wear OS 2.0)
4. Click **Finish** and wait for the initial Gradle sync to complete. You will see errors at this stage — that is expected; the skeleton files will be replaced in the next step.

---

## 2 — Replace skeleton files with project source

With the project open in Android Studio, replace the generated files with those from this repository. The safest method:

1. Close Android Studio.
2. Using Explorer or Git, copy every file from the repo **over** the generated project folder, merging directories. Do **not** delete `gradle/wrapper/gradle-wrapper.jar` — it was generated by Android Studio and is not in the repo.
3. Reopen the project in Android Studio.

### Files that must already exist (generated, not in repo)
- `gradle/wrapper/gradle-wrapper.jar`

### Files that must NOT be committed (machine-specific)
- `local.properties` — see step 3 below.

## 3 — Configure `local.properties`

Copy the provided template and fill in your credentials:

```
copy local.properties.template local.properties
```

Edit `local.properties`:

```properties
sdk.dir=C\:\\Users\\Jamie\\AppData\\Local\\Android\\Sdk
keystore.path=C:/Users/Jamie/Git/.keystore/wled-ble-release.keystore
keystore.password=YOUR_STORE_PASSWORD
keystore.alias=wled-ble
keystore.aliasPassword=YOUR_KEY_PASSWORD
```

**Note on path formats:**
- `sdk.dir` uses the `C\:\\...` form (Android Studio writes it this way automatically).
- `keystore.path` uses forward slashes — the Gradle `file()` call handles them correctly on Windows.

## 4 — Fix `JAVA_HOME` (Windows)

Android Studio ships its own JBR (JetBrains Runtime). Gradle must use this JDK; a stale system `JAVA_HOME` pointing elsewhere causes "toolchain resolution" failures where Gradle tries to download JDK 21 from foojay.io.

The project's `gradle.properties` already contains:
```properties
org.gradle.java.home=C:\\Program Files\\Android\\Android Studio\\jbr
```

If your Android Studio is installed elsewhere, adjust that path.  
You can verify the bundled JBR path inside Android Studio at:  
**File → Settings → Build, Execution, Deployment → Build Tools → Gradle → Gradle JDK**

Additionally, check (and if necessary correct) your system environment variable:

1. Search Windows for **"Edit the system environment variables"**.
2. Under **System variables**, find `JAVA_HOME`.
3. Set it to `C:\Program Files\Android\Android Studio\jbr` (or your AS install path).
4. Restart any open terminals and Android Studio.

## 6 — Check / fix global `gradle.properties`

The machine may have a global Gradle properties file that overrides project settings:

```
%USERPROFILE%\.gradle\gradle.properties
```

If it exists and contains `android.useAndroidX=false`, either:
- Remove that line, or
- Confirm the project-level `gradle.properties` (which sets `android.useAndroidX=true`) takes effect — project-level always wins over global.

## 8 — Connect a physical WearOS device via WiFi ADB

The WearOS emulator cannot communicate with a real BLE peripheral. Use **WiFi ADB** for all BLE testing.

### One-time pairing (WearOS 3+ recommended)

On the watch:
1. **Settings → Developer options** (if not visible: Settings → About → tap "Build number" 7 times).
2. Enable **ADB debugging**.
3. Enable **Wireless debugging**.
4. Tap **"Pair device with pairing code"** — note the IP, port, and 6-digit code shown on screen.

On the development PC:
```bat
cd C:\Users\Jamie\AppData\Local\Android\Sdk\platform-tools

REM Pair once (port and code shown on the watch "Pair" screen):
adb pair <watch-ip>:<pair-port>
REM Enter the 6-digit code when prompted.

REM Connect (port shown on the main "Wireless debugging" screen):
adb connect <watch-ip>:<debug-port>

REM Verify:
adb devices
REM Should show: <ip>:<port>   device
```

### Re-connecting after the watch sleeps
The pairing persists; only the `adb connect` step is needed again if the connection drops.

In Android Studio, the watch now appears in the **device selector** (top toolbar). Select it and click **Run** (▶) to deploy.

## 9 — Debug build — run on device

1. Select your watch in the device selector.
2. Click **Run 'app'** (▶) or press `Shift+F10`.
3. The app installs as `com.jpchurchouse.wledblewear.debug` (the debug suffix from `applicationIdSuffix`).
4. Grant BLE permissions when prompted on the watch.

## 10 — Install the Tile on the watch

Tiles must be added manually by the user:

1. On the watch, swipe right from the main watch face to reach the tile carousel.
2. Scroll to the last tile (the **"+" Add tiles** button).
3. Find **"WLED Control"** in the list and tap it.
4. On first add, the tile shows **"Open app to connect"** because no device is saved in DataStore yet.
5. Open the app, scan, and tap your ESP32 device. On successful connection the device address and name are persisted to DataStore.
6. Swipe back to the tile — it now shows the device name and control buttons.

**Tile refresh:** The tile updates automatically whenever BLE state changes (the ViewModel calls `TileService.getUpdater().requestUpdate()`). You can also long-press the tile and tap **Refresh** if needed.

## 11 — Release AAB build

### Pre-flight checklist
- `local.properties` has correct keystore credentials.
- Version code/name bumped in `app/build.gradle.kts` if this is an update.

### Build
In Android Studio:
1. **Build → Generate Signed Bundle / APK…**
2. Select **Android App Bundle**.
3. Select keystore: `C:\Users\Jamie\Git\.keystore\wled-ble-release.keystore`, alias `wled-ble`.
4. Choose **release** build variant → **Create**.
5. Output: `app/release/app-release.aab`

Or from the command line:
```bat
gradlew bundleRelease
```
Output is at `app\build\outputs\bundle\release\app-release.aab`.

### Play Store upload
1. Open [Play Console](https://play.google.com/console).
2. Navigate to your app → **Production** (or Internal Testing for first upload).
3. Create a new release → upload the `.aab`.
4. Fill in release notes → **Review and roll out**.

> **First-time upload:** The Play Console requires you to opt into Play App Signing. It will manage the upload key from your AAB going forward.

## 12 — BLE testing notes

- The WearOS emulator **cannot** scan for or connect to the ESP32. Use a physical watch for all BLE development.
- Ensure the ESP32 firmware calls `NimBLEAdvertising::addServiceUUID("4fafc201-1fb5-459e-8fcc-c5c9c3319100")` before advertising, otherwise the scan filter will not match it.
- MTU negotiation is initiated by the app (requests 517); NimBLE handles fragmentation server-side. The Android BLE stack reassembles fragmented ATT packets before delivering to `onCharacteristicRead`, so no manual reassembly is needed on the client.
- Use `adb logcat -s BleManager` to watch the GATT operation chain in real time during development.
